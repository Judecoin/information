# The Underlying Protocol Judecoin

> **Date:** January 15, 2021  
> **Category:** News

CryptoNote V2.0 is Judecoin's underlying blockchain protocol. The protocol introduces two technologies: private address technology and ring signature technology, which provide privacy protection for the receiver and the sender respectively.

## 1. Private Address Technology

Private address technology has the characteristics of multiple keys.

The receiver has a private key `b` and a public key `B`, where `B = bG`. `G` is a point on the elliptic curve.

The sender obtains the receiver's public key `B` at the time of transaction and generates the obfuscated address `P`.

The address consists of two parts. One part is generated by hashing the receiver's public key `B` with the addition of a random number `r`, which is only owned by the sender. The other part is the receiver's public key `B`.

At the beginning of the transaction, the sender Alice obtains the public key `B` of the receiver Bob, and the obfuscated address is obtained through a hash and elliptic curve algorithm combined with the random number `r`.

The obfuscated address `P` is generated and recorded by the miner.

The receiver can obtain the random number `r` through the elliptic curve operation on-chain, where `R = rG`.

Then `bR = brG = rbG = rB`.

According to this equation, the receiver may use `r`, the private key `b`, and the public key `B` to generate the same obfuscated address.

The receiver checks all transactions on the blockchain using the private key and public key to confirm whether there are any transactions belonging to them.

Private address technology uses one-time hidden addresses, which greatly enhances the security of the transaction receiver and ensures the information privacy of the receiver.

## 2. Ring Signature Technology

Ring signature technology adds indistinguishable invalid information in the process of information transmission, thus protecting the information privacy of the sender.

In the signature verification process, the validity of the transaction information can be confirmed by verifying the whole signature.

The ring signature is formed by using the private key of the sender and a set of random public keys on the blockchain.

First, the sender obtains `r` obfuscated public keys from the blockchain: `Pi (i = 1, 2, ..., r)`.

Then the sender generates random numbers: `xi (i = 1, 2, ..., r)`.

The sender also generates corresponding values: `yi (i = 1, 2, ..., r)`.

Then the sender selects a random number `v` and generates `ys` through a specific algorithm, in which `v` and `yi` are the parameters.

As the owner of the private key, the sender can calculate `xs` using the private key and `ys`.

The final ring signature consists of the relevant parameters, where `Ps` is the sender's public key.

To verify the signature, the verifier uses the parameters `xi (i = 1, 2, ..., r)` and `xs` through their corresponding one-way trapdoor functions, namely `Pi` and `Ps`, to obtain the corresponding output values `yi (i = 1, 2, ..., r)`.

Finally, `v'` is obtained by the inverse operation of `yi` and `ys` through the above specific algorithm.

By comparison, if `v = v'`, then the ring signature is verified as valid.

If the equation holds, the ring signature is valid.
